# TDD AI-Assisted アプローチ

リバーシ（オセロ）の合法手判定プログラムを**テスト駆動開発（TDD）とAIアシスタント**で実装するアプローチです。

## 概要

このアプローチでは、AIアシスタント（Claude）と協力しながら、ケント・ベックのTDD手法に従って段階的に実装を進めます。

## アプローチの特徴

### 1. 純粋なTDD
- **Red-Green-Refactor サイクル**の厳密な実践
- 最小のテストから開始し、段階的に複雑さを増していく
- テストリスト（TODO.md）で進捗を管理

### 2. AIアシスタントとのペアプログラミング
- Claude（AIアシスタント）と協力して実装
- TDDの原則に従いながら、効率的に開発
- 実装の各ステップでレビューとフィードバック

### 3. 振る舞いベースのテスト
- 具体的な例から始める
- テストは仕様書として機能
- 段階的に機能を追加

## セットアップ

### 前提条件
- Python 3.11.2 以上

### インストール手順

```bash
cd approaches/tdd_ai_assisted
pip install -r requirements.txt
```

## テスト実行

```bash
# 全テスト実行
pytest -v

# 特定のテストファイル実行
pytest test_first_step.py -v
```

## 実装の進め方

### 1. テストリストの作成（TODO.md）
実装すべき機能を小さなステップに分解してリスト化

### 2. 最小のテストから開始
- まず「テストが実行できること」を確認
- 最もシンプルな機能から実装

### 3. Red-Green-Refactor サイクル
1. **Red**: 失敗するテストを書く
2. **Green**: テストをパスする最小限の実装
3. **Refactor**: コードの改善とコミット

### 4. 段階的な実装
- 1x1のボード → 4x4のボード → 8x8のボード
- 簡単な機能から複雑な機能へ
- 各ステップでテストを追加
